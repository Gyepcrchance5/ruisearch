# Word 友好格式规范

适用于需要转换为 .docx 文档的 Markdown 输出。

## 核心规则

- 标准标题层级（H1-H3），避免跳级
- 每个表格必须有完整表头
- 避免使用内联 HTML 标签
- 段落之间留空行
- 使用标准列表格式（`-` 或 `1.`）

## 标题层级

```markdown
# 文档标题
## 一、一级章节
### 1.1 二级章节
```

标题层级必须连续，不跳级（如 H1 后不直接用 H3）。

## 表格

```markdown
| 表头1 | 表头2 | 表头3 |
|-------|-------|-------|
| 内容1 | 内容2 | 内容3 |
```

- 每列必须有表头
- 避免单元格内使用换行符
- 表格前留空行

## 列表

```markdown
- 一级项目
  1. 子项目1
  2. 子项目2
- 一级项目
```

有序列表用 `1.` `2.` `3.`；无序列表用 `- `。

## 图片

```markdown
![图片描述](相对路径/图片.png)
```

图片使用相对路径，方便后续转换。

## 转换命令

Markdown 转 Word 文档：
```bash
python3 tools/md_to_docx.py -i input.md -o output.docx
```

或使用 pandoc：
```bash
pandoc input.md -o output.docx --reference-doc=template.docx
```

## 适用场景

- 需要 .docx 格式的正式文档
- 专利交底书、技术报告、合同文档
- 用户明确要求"Word格式"或"docx格式"
