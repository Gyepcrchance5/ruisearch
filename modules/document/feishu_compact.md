# 飞书紧凑格式规范
适用于需要在飞书文档中保持紧凑排版的 Markdown 输出（**默认格式**）。
## 核心原则
1. 标题与内容之间不留空行
2. 段落之间不留空行
3. 不使用 `---` 分割线
4. 不使用 HTML 标签
5. 加粗仅用于关键词/小标题，正文不加粗
6. 引用块 `>` 仅用于结论摘要、风险提示、注意事项，每章节最多1-2处
## 标题层级
```
# 文档标题
## 一级章节
### 二级章节
```
标题后直接跟内容，不留空行。标题前也不留空行。
## 列表
```
- 一级项目
- 一级项目
```
列表项之间不留空行。无子级缩进需求时统一用 `- `。
## 表格
### 位置规则
- 表格紧跟标题或说明文字，**前面**不留空行
- 表格后跟 `#` 等级标题（`##`、`###`、`####`）：直接跟，**不留空行**
- 表格后跟非标题内容（正文段落、引用块 `>`、加粗子标题、列表）：**在表格最后一行后加一个空行**。不加空行的话，飞书渲染器会把这些内容吞进表格
### 密度控制
- 表格**不超过 4 列**
- 每个单元格内容**不超过 30 字**（含标点）
- 超过 30 字的内容必须用两层结构展开
### 两层结构（表格 + 展开）
当表格需要长文本描述时，禁止塞入单元格。改为：
1. **表格层**：只保留短字段（名称、编号、日期、状态等），每格 ≤ 30 字
2. **展开层**：表格后空一行，再用加粗子标题 + 要点列表展开详细内容
示例：
```
| 编号 | 名称 | 日期 | 状态 |
| --- | --- | --- | --- |
| A-001 | 技术方案A | 2025-03 | 已验证 |
| A-002 | 技术方案B | 2025-01 | 待验证 |

- **A-001 技术方案A：** 采用X架构，性能提升30%
- **A-002 技术方案B：** 引入Y算法，精度提升至99.5%
```
### 禁止事项
- 禁止在单元格内使用嵌套列表
- 禁止单个单元格超过 30 字
- 禁止在单元格内用分号串联长句
- **禁止在表格单元格内使用加粗**（飞书渲染异常）

## 引用块
```

```
使用场景：结论摘要、风险提示、数据置信度说明。每章节最多1-2处，不宜过度使用。

引用块放在表格后面时，表格和引用块之间必须空一行，否则飞书会把引用块吞入表格。

引用块内可嵌列表：
```
> 注意事项：
> - 数据来源单一，置信度标注为"待验证"
> - 部分参数为估算值
```

## 说明文字与表格的关系
说明文字在表格上方，表格作为支撑数据在下方：
```
### 3.1 品牌A
该品牌在细分市场占有率领先，技术路线以X方案为主。
| 产品 | 价格 |
| --- | --- |
| 产品X | 5万起 |
```
## 子品牌/子分类
用加粗文字标识，紧跟表格。多个子品牌连续排列时，表格和下一个子品牌之间空一行：
```
**子品牌A（集团X旗下）**
| 产品 | 价格 |
| --- | --- |
| 产品Y | 15万起 |

**子品牌B（集团Y旗下）**
| 产品 | 价格 |
| --- | --- |
| 产品Z | 20万起 |
```

## 生成后自检清单
**每次输出飞书紧凑格式文档后，必须逐条执行以下检查。不通过的不允许交付。**
### 表格检查
- [ ] 扫描全文每一张表格：所有单元格内均不包含 `**`（飞书渲染异常）
- [ ] 每张表 ≤ 4 列
- [ ] 每个单元格 ≤ 30 字（含标点）
- [ ] 无嵌套列表、无分号串联长句
### 位置检查
- [ ] 表格后紧跟 `#` 标题时：中间无空行
- [ ] 表格后紧跟非标题内容（正文、引用块 `>`、加粗子标题、列表）时：中间有且仅有一个空行
- [ ] 不存在表格最后一行后无空行直接跟非标题内容的情况（会被飞书吞入表格）
### 全局检查
- [ ] 文档中无 `---` 分割线
- [ ] 无 HTML 标签
- [ ] 标题与内容间无空行，段落间无空行，列表项间无空行（表格后的隔离空行除外）
- [ ] 加粗仅用于关键词/小标题，未整段加粗
- [ ] 引用块每章节 ≤ 2 处
