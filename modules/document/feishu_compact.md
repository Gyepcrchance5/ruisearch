# 飞书紧凑格式规范
适用于需要在飞书文档中保持紧凑排版的 Markdown 输出（**默认格式**）。
## 核心原则
1. 标题与内容之间不留空行
2. 段落之间不留空行
3. 不使用 `---` 分割线
4. 不使用 HTML 标签
5. 加粗仅用于关键词/小标题，正文不加粗
6. 引用块 `>` 仅用于结论摘要、风险提示、注意事项，每章节最多1-2处
## 标题层级
```
# 文档标题
## 一级章节
### 二级章节
```
标题后直接跟内容，不留空行。标题前也不留空行。
## 列表
```
- 一级项目
- 一级项目
```
列表项之间不留空行。无子级缩进需求时统一用 `- `。
## 表格
### 位置规则
- 表格紧跟标题或说明文字，前面不留空行
- **表格后面只能跟 `#` 等级标题**（如 `###`、`####`），禁止使用 `**加粗**` 做子标题。飞书渲染器会把紧跟表格的加粗文本吞进表格。
- 表格后直接跟标题，不留空行。
- **正文段落必须放在表格前面，**不能紧跟在表格最后一行之后（飞书会把正文吞进表格渲染）
### 密度控制
- 表格**不超过 4 列**
- 每个单元格内容**不超过 30 字**（含标点）
- 超过 30 字的内容必须用两层结构展开
### 两层结构（表格 + 展开）
当表格需要长文本描述时，禁止塞入单元格。改为：
1. **表格层**：只保留短字段（名称、编号、日期、状态等），每格 ≤ 30 字
2. **展开层**：表格下方用加粗子标题 + 要点列表展开详细内容
示例：
```
| 编号 | 名称 | 日期 | 状态 |
| --- | --- | --- | --- |
| A-001 | 技术方案A | 2025-03 | 已验证 |
| A-002 | 技术方案B | 2025-01 | 待验证 |
- **A-001 技术方案A：** 采用X架构，性能提升30%
- **A-002 技术方案B：** 引入Y算法，精度提升至99.5%
```
### 禁止事项
- 禁止在单元格内使用嵌套列表
- 禁止单个单元格超过 30 字
- 禁止在单元格内用分号串联长句
- **禁止在表格单元格内使用加粗**（飞书渲染异常）
## 引用块
```
> 核心发现：该方案成本降低约15-20%。
```
使用场景：结论摘要、风险提示、数据置信度说明。每章节最多1-2处，不宜过度使用。
引用块内可嵌列表：
```
> 注意事项：
> - 数据来源单一，置信度标注为"待验证"
> - 部分参数为估算值
```
## 说明文字与表格的关系
说明文字在表格上方，表格作为支撑数据在下方：
```
### 3.1 品牌A
该品牌在细分市场占有率领先，技术路线以X方案为主。
| 产品 | 价格 |
| --- | --- |
| 产品X | 5万起 |
```
## 子品牌/子分类
用加粗文字标识，紧跟表格：
```
**子品牌A（集团X旗下）**
| 产品 | 价格 |
| --- | --- |
| 产品Y | 15万起 |
```
## 完整示例
```
# 文档标题
## 一、核心概念
### 1.1 定义
说明文字：
- 要点1
- 要点2
### 1.2 对比
两种类型各有优劣，A类型适用于高频场景，B类型适用于低成本场景。
| 类型 | 特点 |
| --- | --- |
| A类型 | 高频适用 |
| B类型 | 低成本 |
## 二、产品清单
### 2.1 品牌A
| 产品 | 价格 | 备注 |
| --- | --- | --- |
| 产品1 | 10万起 | 官网数据 |
| 产品2 | 20万起 | 经销商报价 |
### 2.2 品牌B
| 产品 | 价格 | 备注 |
| --- | --- | --- |
| 产品3 | 30万起 | 官网数据 |
## 三、总结
> 核心结论：A品牌性价比最优，B品牌技术领先。
| 统计项 | 数据 |
| --- | --- |
| 总数 | X个 |
- 结论1
- 结论2
```
