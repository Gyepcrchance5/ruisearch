# 标准 Markdown 格式规范

适用于 GitHub README、通用技术文档、非飞书环境的 Markdown 输出。

## 核心规则

- 遵循 CommonMark 规范
- 允许 `---` 分割线
- 允许 `>` 引用块
- 标题前后可留空行
- 段落之间建议留空行提升可读性
- 允许使用 HTML 标签（如 `<details>`、`<br>`）

## 标题层级

```markdown
# 文档标题

## 一级章节

### 二级章节
```

标题前后留空行，符合标准 Markdown 习惯。

## 列表

```markdown
- 一级项目
  - 子项目
  - 子项目
- 一级项目
```

支持多级嵌进缩进。

## 表格

```markdown
| 表头1 | 表头2 | 表头3 |
|-------|-------|-------|
| 内容1 | 内容2 | 内容3 |
```

列数和内容长度无特殊限制。

## 代码块

使用三反引号包裹，指定语言高亮：

```markdown
```python
print("hello")
`` `
```

## 引用

```markdown
> 引用内容
```

## 适用场景

- GitHub 项目文档
- 通用技术文档
- 非飞书平台的 Markdown 输出
- 用户明确要求"标准 Markdown"或"普通格式"
